# Interview Intel - 事实验证系统

## 概述

本项目已启用了**事实验证协议**（Fact Verification Protocol），确保Claude Code在处理面试准备相关任务时，严格按照用户真实经历执行，**绝不编造或假设任何信息**。

---

## 系统组成

### 1. 核心协议文件
**文件位置**: [`.claude/FACT_VERIFICATION_PROTOCOL.md`](.claude/FACT_VERIFICATION_PROTOCOL.md)

**内容**:
- 零容忍原则：禁止编造用户信息
- 强制验证流程：数据源验证 → 关键事实确认 → 生成前校验
- PDF读取失败处理协议
- 用户确认机制
- 事实核查清单

### 2. 自动触发机制
**文件位置**: [`.claude/settings.local.json`](.claude/settings.local.json)

通过hooks机制，当您提交以下关键词相关请求时，自动触发验证：
- `interview` / `resume` / `jd` / `job` / `application`
- `面试` / `简历`

---

## 如何使用

### 作为用户

**1. 提供简历时**：
```
建议格式（按优先级）：
✅ 纯文本简历 (.txt, .md)
✅ 简历截图 (PNG, JPG) - 我可以读取图片中的文字
⚠️  PDF文件 - 如果读取失败，我会明确告知
❌  不要让我"猜测"或"补充"您的经历
```

**2. 当我发现数据问题时**：
我会：
- 🛑 立即停止任务
- ⚠️ 明确告知问题所在
- 📋 要求提供可用的数据格式

**3. 生成内容前**：
我会：
- ✅ 向您确认关键信息
- ✅ 使用"待确认"标记不确定的内容
- ✅ 引用简历中的原文

---

## 您的权利

### 数据准确性保证

您有权要求：
1. **数据来源说明** - "这信息从哪里来的？"
2. **事实核查** - "请确认这是否在简历中"
3. **即时修正** - "这是错的，请立即修正"

### 当发现错误时

请立即告诉我：
```
❌ "这不对，我没有在字节工作过"
✅ "我的简历中写的是贝壳，不是字节"
```

我会：
1. 立即停止
2. 承认错误
3. 重新读取正确的数据
4. 修正所有相关内容

## 常见问题

### Q: 为什么需要这个系统？
**A**: 我之前犯过错误，编造了用户没有的经历。这个系统确保不再发生类似错误。

### Q: 这个系统100%可靠吗？
**A**: 这是一个保障机制，但不是完美的。如果您发现任何不准确之处，请立即告诉我。

### Q: 我需要做什么？
**A**:
1. 提供清晰可读的简历格式
2. 当我向您确认信息时，仔细核对
3. 发现任何错误立即指出

### Q: 可以关闭这个系统吗？
**A**: 强烈不建议！这个系统保护您的数据准确性。如果一定要关闭，请编辑 `.claude/settings.local.json` 删除hooks部分。

## 技术细节

### Hook触发条件

```json
"matcher": "(?i)(interview|resume|jd|job|application|面试|简历)"
```

匹配包含以下任一关键词的请求：
- interview, resume, jd, job, application
- 面试, 简历

### 验证提示词

```
CRITICAL: Before processing any interview/resume related request:

1. Check if FACT_VERIFICATION_PROTOCOL.md exists
2. If it exists, read it and strictly follow all verification steps
3. Never assume or fabricate user's work experience
4. If resume PDF fails to read, explicitly inform the user
5. All user personal info must be verified before use
```
