# Interview Intel - 面试准备助手 v1.0.0

使用 Interview Intel v1.0.0 系统一键生成完整的面试准备包。

## ⚡ 核心特性

**事实验证协议 (Fact Verification Protocol)** - 确保生成内容100%基于真实简历：

✅ **零容忍原则**：绝不编造、假设或推断用户的个人信息、工作经历、项目成果
✅ **三阶段验证**：简历读取 → 信息确认 → 事实核查清单
✅ **自动触发**：检测到面试相关请求时自动启用验证流程

## 使用方式

直接告诉我你想应聘的公司和职位，我会自动生成完整的面试准备包。

**示例**：
```
我想应聘 MiniMax 的 B端大模型产品经理职位
JD: [粘贴 JD 内容或提供文件路径]
简历: resumes/王蕾-AI 产品经理V1.0.pdf（可选）
```

## 功能说明

此命令会自动完成以下工作：

1. **严格事实验证** ⚡ NEW
   - 读取 FACT_VERIFICATION_PROTOCOL.md 并严格遵守
   - 使用 pdfplumber 成功提取简历文本
   - 确认关键信息（公司、项目、数据）后才继续
   - 所有生成内容包含事实声明和核查清单

2. **运行脚本生成框架** - 执行 interview-intel/scripts/all_in_one.py 生成 5 个标准文件
3. **读取简历提取信息** - 自动从 resumes/ 目录的 PDF 简历提取核心经历
4. **主动询问缺失信息** - 如需要会询问你（如公司官网、产品熟悉度等）
5. **网络搜索补充信息** - 获取公司最新信息、竞争对手、行业趋势
6. **AI 智能填充内容** - 填充所有内容，生成真实的 STAR 案例和面试话术

## 输出文件

生成 5 个完整的 Markdown 文件（保存在 **companies/[公司名]-[职位]-[候选人名字]/**）：

**命名规范**：`companies/贝壳-MR产品经理-李承润/`
- 支持同一公司多个岗位：`贝壳-MR产品经理-张三/` 和 `贝壳-AI算法工程师-李四/`
- 支持多人面试同一岗位：`贝壳-MR产品经理-张三/` 和 `贝壳-MR产品经理-李四/`

1. **01_company_intel_brief.md** - 公司背景业务信息
2. **02_resume_jd_matching.md** - 简历-JD 匹配分析
3. **03_interview_prep_report.md** - 面试准备报告（三轮话术）
4. **04_icebreaker_messages.md** - 破冰开场白
5. **05_final_analysis_report.md** - 最终分析报告和行动计划

## 参数说明

**必需参数**：
- `company` - 公司名称
- `role` - 目标职位
- `jd` - JD 内容或文件路径

**可选参数**：
- `resume` - 简历文件路径（默认读取 resumes/ 目录）
- `industry` - 所属行业
- `keywords` - 关键词（逗号分隔）
- `achievement` - 核心成就描述
- `years` - 工作年限

## 工作流程（v1.0.0 事实验证版）

```
用户请求 → 触发事实验证协议 → 读取简历 → 提取并确认关键信息 → 生成内容 → 事实核查清单
```

### 步骤 1: 自动触发验证
检测到关键词（interview/resume/jd/job/application/面试/简历）时自动启用验证流程

### 步骤 2: 简历读取与验证
- 使用 pdfplumber 成功提取简历文本（不使用 Read 工具）
- 提取关键信息：姓名、公司、项目、数据
- 向用户确认关键信息的准确性

### 步骤 3: 内容生成
- 所有内容严格基于确认的简历信息
- 每个文件包含"事实声明"章节
- 标记不确定内容为"⚠️ 需补充"或"⚠️ 需面试时补充"

### 步骤 4: 事实核查清单
生成文件包含核查清单：
- [x] 公司名称与简历一致
- [x] 项目名称与简历一致
- [x] 数据有简历依据
- [x] 所有陈述都有事实依据

## 注意事项

**事实验证**：
- ✅ 所有公司名称、项目名称、数据均来自简历
- ✅ 未添加任何简历中没有的经历
- ⚠️ 标记为"待补充"的内容需要您后续完善
- ⚠️ 如发现任何不准确之处，请立即告知

**使用提示**：
- 如果简历未指定，会尝试从 resumes/ 目录自动查找
- 如果信息不足，会在对话过程中主动询问你
- 生成的所有文件保存在 **companies/[公司名]-[职位]-[候选人名字]/** 目录
- 命名示例：`贝壳-MR产品经理-李承润/`
- 首次使用会自动创建必要的目录结构
- PDF 读取失败时会明确提示，不会继续使用编造数据
